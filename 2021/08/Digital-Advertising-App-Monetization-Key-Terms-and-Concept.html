互联网数字营销系列-广告与流量变现指标

日 01 八月 2021

围绕广告业务工作了很多年，一直有整理广告与变现指标的冲动，每次都成功的抑制住了这种冲动；最近在整理过去的产品资料，花些时间整理一份，为今后文档工作做好准备。

DAU

daily active user 当前的活跃用户数量，包含当天新注册用户+老用户

MAU

月活跃用户数量

Impression

曝光，指广告素材的完成显示；视频广告的曝光标准可按实际播放的时长折算

click

点击，批广告素材的点击并产生跳转

CTR

Click-through rate, the ratio of clicks to impressions. 点击率，用户点击广告素材的数量与广告曝光数量比例。

CPC

Cost Per Click 每单次广告点击的成本

CVR

广告转化量，生产真实转化的数量与广告点击数量的比例；业务不同，转化定义不同，如下载，安装，使用，充值行为等

CVR =转化量/点击量

CPA

Cost Per Action 每单次转化行为的成本，业务不同，转化定义不同，但通过CVR计算CPA时，使用相同的转化定义

CPC = 投放成本/转化量=（CPC*点击量）/（CVR*点击量）=CPC/CVR

CPM

Cost Per Mille, cost per 1000 impressions 每千次广告展示单元的成本

CPT

Cost Per thousand unique impressions 每千次去重用户后的广告展示成本/价格；该指标近几年逐渐被边缘化。

eCPM

每千次广告展示的价格 用于评估广告预估收益，广告能给媒体平台带来的价值，叧一个来源的eCPM值越高，广告系统就优先展示哪一来源的广告。

eCPM的计算方式，不同应用场景下有不同的计算方式

方式1：eCPM = CTR*CPC
方式2：eCPM = (当日广告收益/广告曝光总数)*1000
方式3：eCPM = 1000*pCTR* CPC  
方式4：eCPM = 1000*pCTR*pCVR*CPC # 竞价业务中使用 
方式5：eCPM = 1000*pCTR*pCVR*CPA # 竞价业务中使用
备注：pCTR指预计点击率（点击量/展示量），pCVR指预计转化率（转化量/点击量）

eCPI

Effective Cost Per Install

oCPC

Optimized Cost Per Click 一种优化投放的方式与计费方式，广告平台将投放目标转化为以优化转化为主，对最有可能发生转化的用户投广告，并最终按CPC结算收费； 对于广告主来说，相比于普通的CPC模式，在同样计价方式下oCPC的转化率理论上会更高，因为普通的CPC并没有将后续的转化做为目标加入考核。

相比于CPA的考核方式，oCPC以点击为主，考核标准上难度更大，但单个安装的费用理论上应该低于CPA方式 ，但总投放费用模式增大；

oCPM

optimized CPM

bidding

竞价出价 DSP中对ADX发起的竞价请求做出合理价格预估的过程 ADX/RTB竞价业务中，通常是按CPM方式进行，部分系统支持CPC或CPA的方式出价,实际操作是将CPC或CPA转化成CPM，本质还是CPM方式。

具体出价金额需需要计算pCTV/pCVR，并预估eCPM，操作如下

方式3：eCPM = 1000*pCTR* CPC  
分析：CPC值由广告主在投放计划中设定，eCPM预测的准确与否就直接取决于系统对CTR的预测是否准确的，预估模型与算法的核心就转化为对广告CTR的预估。


方式5：eCPM = 1000*pCTR*pCVR*CPA # 竞价业务中使用
分析：CPA是固定值，pCRT和pCVR均需要预估得出，将出价预估的问题，转为计算pCTR和pCVR。

难点：实际执行中，CTR的预估常常不准确，对于一个新的请求来说，缺少数据沉淀，同时现有数据的离散分布，均导致很难预估一个合理的CTR值

广告渗透率

广告渗透率即广告展示所覆盖的用户数占活跃用户总数的比例。 渗透率越高，说明广告覆盖到的用户量级越广泛，广告曝光量也越高，因此越有助于App提升总的广告收益。

Fill Rate

广告填充率，广告请求数与广告内容成功曝光的比值

当日收益：

当日，在各广告平台的流量售卖所带来的实际收益

单日收益

计算方式为当日成功播放的广告的总数与每个广告的价格的乘积；

单日收益 = 当日广告的Impressions*eCPM/1000

ARPU

ARPPU ARPDAU Average Revenue Per User 以上只是时间周期不同，定义一致 平均每用户收入，指定时间周期内，例如设定为每日，指标为单用户每日价值；如周期设为每月，则指单用户的每月价值；

就是ARPU 值=总收入 / 活跃用户数（指定周期内）

ARPU=单日总收益/当日DAU

LT

生命周期

LTV

Lifetime Values

用户生命周期价值，代表营销渠道获取的新用户，在该用户完整生命周期内给APP创造的价值与收入总额。 LTV越高，App流量变现的价值越高，该指标用于衡量App广告的变现收益能力，是APP运营中一个重要指标。

实际应用中，LTV指标更多的用于游戏行业，主要是衡量获客（买量）和变现（卖量）之间的关系。 对于通过广告变现的开发者而言，了解App的LTV值同样对流量变现起到重要作用。它可以更好的评估App用户价值，计算回报周期、验证盈利模式，获得更高收益。 LTV是一个带时间概念的值，在计算的时候需求指定具体的天数或周期

LTV的计算方式比较复杂，简单的方式说明，用新增用户在留存期内的总收益/新增用户数，其中存留周期可以是7天，15天或30天。

LTV(n) 多天用户价值的总合

LTV(n) = 当日的新增用户在往后n天内的总收益 / 当日的新增用户数
LTV(n) = ∑(第一日价值/收益 + 每二日价值/收益 + 每三日价值/收益 + 第N日价值/收益)

ROI

投入产出比

ROI = LTV(n)/买量投入

LT与LTV关系

LT拉长，LVT会累计增加

eCPM与LTV关系

eCPM提高，LTV会相应提升

回本周期

当LTV值大于买量投入的成本时，业务进入回本周期； 对于APP下载安装类应用，买量投入按eCPI值计算
